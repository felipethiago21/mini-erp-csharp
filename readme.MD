## рџЏ—пёЏ Arquitetura

Camadas inspiradas em Clean Architecture / Onion Architecture:

- **MiniErp.Domain** вЂ” entidades e regras de negГіcio intrГ­nsecas (`Produto`, `Cliente`, `Venda`, `ItemVenda`).
- **MiniErp.Application** вЂ” casos de uso: DTOs, interfaces (repositГіrios, unit of work, services), services, validators (FluentValidation) e exceptions de negГіcio.
- **MiniErp.Infrastructure** вЂ” Entity Framework Core (`MiniErpContext`), implementaГ§ГЈo dos repositГіrios, `UnitOfWork` e migrations.
- **MiniErp.Api** вЂ” ASP.NET Core Web API: controllers, middleware de exceГ§Гµes, Swagger, CORS e configuraГ§ГЈo de DI.
- **MiniErp.Console** вЂ” aplicativo console original, agora consumindo as mesmas camadas de Application/Infrastructure via injeГ§ГЈo de dependГЄncia (mantido como demonstraГ§ГЈo histГіrica).
- **MiniErp.Tests** вЂ” testes xUnit dos services de Produto e Venda.
- **MiniErp.Web** вЂ” SPA em React + TypeScript (Vite) que consome a API via HTTP, sem acoplamento a nenhuma camada .NET.

Regra importante: os **controllers nГЈo contГЄm regra de negГіcio** вЂ” apenas orquestram DTOs e chamam os services da camada Application. As regras vivem nas entidades (`Domain`) e nos services (`Application`).
